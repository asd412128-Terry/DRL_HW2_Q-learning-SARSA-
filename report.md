# Q-learning 與 SARSA 演算法實驗分析報告

## （三）結果分析

### 1. 學習表現
- **累積獎勵（Total Reward）曲線**：從實驗輸出的 `result_sample.jpg` 獎勵曲線圖中可以看出，兩者的學習曲線在初期都有較大幅度的波動。然而隨著訓練回合數增加，SARSA 的平均累積獎勵迅速上升並收斂在較高的數值（約 -20 左右），而 Q-learning 的平均累積獎勵則維持在較低的水準（約 -40 到 -50 之間）。
- **收斂速度**：在「在線表現（Online Performance）」上，SARSA 收斂到穩定獎勵的速度比 Q-learning 更快，因為它很快就學會了避開懸崖邊緣，大幅減少了因隨機探索而掉入懸崖所帶來的巨大懲罰（-100）。

### 2. 策略行為
- **最終學習到的路徑視覺化**：根據輸出的 `cliff.jpg`：
  - **Q-learning** 學習到的是一條**緊貼懸崖邊緣**的最短路徑（從起點出發後直接往右沿著懸崖邊緣走到終點）。
  - **SARSA** 學習到的則是一條**遠離懸崖**的安全路徑（先往上走遠離邊緣，接著再往右，最後才往下走向終點）。
- **冒險與保守分析**：Q-learning 的行為極度**冒險**，為了追求全局步數最少的最佳解，它完全不顧慮探索時掉入懸崖的風險；相對地，SARSA 的行為非常**保守**，它寧願多走幾步路來換取絕對的安全性，避免因隨機探索造成的失誤。

### 3. 穩定性分析
- **學習過程中的波動程度**：Q-learning 在訓練過程中的表現波動非常大，即使在訓練後期，它的單次回合獎勵依然常常因為一次隨機探索（$\epsilon$-greedy）而掉入懸崖，導致獲得 -100 的極低分。SARSA 則在找到安全路徑後，波動程度大幅降低，表現非常穩定。
- **探索（Exploration）對結果的影響**：在本實驗中，探索機制（$\epsilon=0.1$）是導致兩者行為差異的關鍵。Q-learning 在更新 Q 值時「假設」未來會做出最佳動作，忽略了 $\epsilon$ 探索帶來的隨機失誤；而 SARSA 則是在更新時「誠實地」將 $\epsilon$ 探索帶來的隨機動作納入考量，因此它意識到走在懸崖邊緣有 10% 的機率會掉下去，進而選擇了遠離邊緣的安全路線。

## 五、理論比較與討論

在強化學習中，這兩種時間差分（Temporal Difference）演算法的根本差異在於其策略更新機制的設計：

- **Q-learning 為離策略（Off-policy）方法**：
  其更新公式基於「下一狀態的最佳可能行動」—即 $max_a Q(S', a)$。這代表在更新價值函數時，無論 Agent 當下實際上是依照什麼策略（如 $\epsilon$-greedy）在探索世界，它總是「貪婪地」拿下一個狀態中價值最高的動作來計算目標值。因此，Q-learning 傾向學習到理論上的最優策略（最短路徑），但在訓練過程中，因為實際的探索行為會不小心偏離這條危險的最佳路徑，導致訓練時期的風險較高且平均回報較低。

- **SARSA 為同策略（On-policy）方法**：
  其更新公式基於「實際採取的行動」—即 $Q(S', A')$。這代表 Agent 是根據當前正在執行的策略（例如 $\epsilon$-greedy）所實際抽樣出的下一個動作來進行更新。因為更新機制忠實地反映了探索策略的影響，Agent 會「意識到」自己偶爾會手滑（隨機探索），因此 SARSA 傾向學習在實際探索策略下較為安全、穩定的行為。

---

## 六、結論

總結 Q-learning 與 SARSA 在本次 Cliff Walking 實驗中的差異：

1. **哪一種方法收斂較快？**
   - 如果以「訓練期間的累積獎勵（Online Reward）」來看，**SARSA 收斂較快**。它能迅速找到一條避開嚴重懲罰的安全路徑，並使平均獎勵穩定下來。
   - 如果以「找到全局最佳解（Optimal Policy）」來看，**Q-learning** 能夠找到真正的最短路徑（即便訓練時表現不佳），而 SARSA 找到的則是考量了 $\epsilon$ 的次佳解。

2. **哪一種方法較穩定？**
   - **SARSA 較為穩定**。因為它的策略內建了對隨機探索的風險迴避，訓練過程的波動較小，平均回報也明顯高於 Q-learning。

3. **在何種情境下應選擇 Q-learning 或 SARSA？**
   - **選擇 Q-learning 的情境**：當訓練環境處於完美的虛擬模擬器中，且「訓練過程中的試錯成本極低或為零」時。在此情境下，我們只在乎最終訓練出來的 Agent 能否找出理論上最完美的極限操作（如棋盤遊戲或無物理損傷的電玩遊戲）。
   - **選擇 SARSA 的情境**：當「訓練過程中的錯誤成本非常高昂」時。例如訓練真實世界的自駕車、機器人或進行醫療決策。在這些情境下，我們不允許系統在學習過程中為了追求極限的最優解而頻繁發生災難性失誤，此時 SARSA 安全穩定的保守策略將是更好的選擇。
