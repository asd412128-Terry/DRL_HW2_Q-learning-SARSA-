# Q-Learning 與 SARSA 演算法之比較研究規格表

## 1. 演算法基礎
- **Q-Learning**：離線學習 (Off-policy)。在更新 Q 值時，不論當前使用什麼策略而產生動作，它總是貪婪地選擇下一個狀態中具有最大 Q 值的動作來進行更新。
- **SARSA**：在線學習 (On-policy)。在更新 Q 值時，會根據目前使用的策略 (例如 ε-greedy) 實際選出的下一個動作來進行更新。

## 2. 演算法更新規則
- **Q-Learning**:
  `Q(S, A) = Q(S, A) + α * [R + γ * max(Q(S', a)) - Q(S, A)]`
- **SARSA**:
  `Q(S, A) = Q(S, A) + α * [R + γ * Q(S', A') - Q(S, A)]`

## 3. 在 Cliff Walking (崖壁行走) 環境中的特性展現
- **Q-Learning**：因為更新時假設未來總是採取最佳解，因此會學習到**最佳路徑 (緊貼懸崖邊緣)**。這導致在訓練期間仍保留隨機探索時，偶爾會掉入懸崖，訓練時的平均獎勵較低、風險高。
- **SARSA**：因為更新時會將未來的探索動作風險考量進去。因此，它會學到一條**偏離懸崖的安全路徑**（避免因隨機探索掉入懸崖），訓練時的平均獎勵較高、波折較少。

## 4. 總結比較表
| 比較項目 | Q-Learning (Q學習) | SARSA |
| :--- | :--- | :--- |
| **策略類型** | 離線策略 (Off-policy) | 在線策略 (On-policy) |
| **目標策略更新** | 選擇下個狀態的最高 Q 值 `max_a Q(s',a)` | 根據實作策略採取的下個動作 `Q(s',a')` |
| **風險承擔** | 追求全局最優，不考慮探索風險 | 考量當前探索風險，較為保守 |
| **懸崖環境路徑** | 緊貼懸崖邊緣 (高風險但最短) | 遠離懸崖緣 (安全但較遠) |
| **收斂性** | 只要所有狀態動作對都被不斷訪問，保證收斂於最佳 Q* | 收斂於依賴探索率 ε 的次佳解 |
