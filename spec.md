# Q-learning 與 SARSA 演算法之比較研究規格書

## 1. 專案概述
本專案旨在實作強化學習中兩種經典的時間差分 (Temporal Difference, TD) 演算法：**Q-learning** 與 **SARSA**，並在「崖壁行走」(Cliff Walking) 環境中比較兩者的學習行為與收斂結果。

## 2. 環境設定 (Cliff Walking Environment)
- **網格大小**: 4 (高) x 12 (寬)。
- **起點**: 左下角 (3, 0)。
- **終點**: 右下角 (3, 11)。
- **懸崖區域**: 網格底部的中間部分，即座標 (3, 1) 到 (3, 10)。
- **動作**: 上 (Up), 下 (Down), 左 (Left), 右 (Right)。
- **獎勵機制**:
  - 每走一步：-1。
  - 掉入懸崖：-100，並強制返回起點 (3, 0)。
  - 到達終點：回合結束。

## 3. 演算法規格
### 3.1 Q-learning (離線學習 / Off-policy)
- **更新規則**: 
  `Q(S, A) ← Q(S, A) + α * [R + γ * max_a Q(S', a) - Q(S, A)]`
- **特性**: 總是假設未來會採取最佳動作（具有最大 Q 值的動作），因此能學習到全局最佳路徑（緊貼懸崖邊緣，步數最少）。但在訓練過程中，由於 ε-greedy 策略的隨機探索，容易掉入懸崖，導致訓練期間的平均獎勵較低。

### 3.2 SARSA (在線學習 / On-policy)
- **更新規則**:
  `Q(S, A) ← Q(S, A) + α * [R + γ * Q(S', A') - Q(S, A)]`
- **特性**: 更新時考量了當前策略（如 ε-greedy）實際選擇的下一個動作 `A'`，因此將探索所帶來的風險（掉入懸崖）納入考量。最終會學到一條較為保守、遠離懸崖的安全路徑，訓練期間平均獎勵較高。

## 4. 參數設定
- **學習率 (α, Alpha)**: 0.1
- **折扣因子 (γ, Gamma)**: 0.9
- **探索率 (ε, Epsilon)**: 0.1
- **訓練回合數 (Episodes)**: 500
- **獨立實驗次數 (Runs)**: 50

## 5. 輸出產物 (Artifacts)
1. **獎勵曲線圖 (result_sample.jpg)**: 顯示 Q-learning 和 SARSA 演算法在訓練過程中的每回合獲得的獎勵 (Reward per Episode)。SARSA 的曲線應在初期與平穩期高於 Q-learning。
2. **路徑圖 (cliff.jpg)**: 視覺化兩個演算法最終學習到的路徑。Q-learning 應為緊貼懸崖的最佳路徑，SARSA 應為遠離懸崖的安全路徑。
